# refactor: decouple-scanner-from-watcher

Status: **Ready for implementation.** All design decisions resolved. No open questions.

---

## Context

The `scanner.Scanner` interface exists with a mock generated by mockery, but the only adapter
(`scanner.Service`) is self-constructed inside `watcher.New()` — the mock is uninjectable.
The `scanner.Service` struct stores a `*slog.Logger` field that neither `KnownFilenames()` nor
`ScanAll()` ever reads. This is the identical pattern already fixed for `parser.Parser`
(made stateless, centralized in `app.go`) and `docker.Docker` (injected through constructors).

The `watcher.Service` stores the scanner as `scanner.Scanner` interface — the seam exists.
The problem is ownership: construction happens inside the consumer, not at the root.

---

## Decision Log

All decisions were reached through a design interview. Do not re-open them without a specific technical reason.

| # | Decision |
|---|---|
| 1 | `scanner.Service` becomes stateless — remove the `logger` field and `New(logger)` param. |
| 2 | `watcher.New()` gains a `scanner.Scanner` parameter; internal `scanner.New(logger)` call is removed. |
| 3 | `app.go` creates `scanner.New()` once and passes it to `watcher.New()`. |
| 4 | Scanner test file removes `newTestLogger()` and its associated imports (`bytes`, `log/slog`). |
| 5 | Watcher test file keeps its `newTestLogger()` (watcher uses its logger for fsnotify errors) and passes `scanner.New()` as the new param. |

---

## Implementation Steps

### Step 1 — Make `scanner.Service` stateless

Edit `internal/services/scanner/service.go`:

- Remove the `logger *slog.Logger` field from `Service`.
- Change `New(logger *slog.Logger) Service` to `New() Service`, returning `Service{}`.
- Remove `"log/slog"` from imports.

**Verify:** `go build ./internal/services/scanner/...` passes (test file will fail — that's next).

### Step 2 — Update scanner test

Edit `internal/services/scanner/service_test.go`:

- Replace all `scanner.New(newTestLogger())` with `scanner.New()`.
- Delete the `newTestLogger()` function.
- Remove `"bytes"` and `"log/slog"` from imports.

**Verify:** `go test ./internal/services/scanner/...` passes.

### Step 3 — Inject scanner into `watcher.New()`

Edit `internal/services/watcher/service.go`:

- Change signature from `func New(dir string, logger *slog.Logger, extraFile string) (Watcher, error)` to `func New(dir string, logger *slog.Logger, extraFile string, sc scanner.Scanner) (Watcher, error)`.
- Remove line 73: `sc := scanner.New(logger)`.
- The `import "github.com/ma-tf/ogle/internal/services/scanner"` stays (needed for `scanner.Scanner` type in signature).

**Verify:** `go build ./internal/services/watcher/...` fails (test file needs update — next step).

### Step 4 — Update watcher test

Edit `internal/services/watcher/service_test.go`:

- Add `"github.com/ma-tf/ogle/internal/services/scanner"` to imports.
- Replace every `watcher.New(dir, newTestLogger(), "")` call with `watcher.New(dir, newTestLogger(), "", scanner.New())`.
- Keep the existing `newTestLogger()` (watcher uses it for fsnotify error logging).

**Verify:** `go test ./internal/services/watcher/...` passes.

### Step 5 — Wire in `app.go`

Edit `internal/app/app.go`:

- Add `"github.com/ma-tf/ogle/internal/services/scanner"` to imports.
- Before the `watcher.New()` call, create `scanSvc := scanner.New()`.
- Change `watcher.New(watchDir, log, projectFile)` to `watcher.New(watchDir, log, projectFile, scanSvc)`.

**Verify:** `go build ./... && go vet ./... && golangci-lint run ./... && go test ./...` all pass.

## Out of Scope

- Rewriting watcher tests to use `MockScanner` instead of real filesystem events. The injectable seam is now open for that, but writing those tests is a separate activity.
- Generalizing the "service interface injection" pattern beyond Scanner/Watcher. The established convention is clear: `app.go` creates service instances, injects them through constructors.
- Adding or removing any interfaces. The existing `scanner.Scanner` and `watcher.Watcher` interfaces are unchanged.
